linux lamp非主流搭建

发表于 2016-03-01 | 分类于 linux

1. 环境和软件

  1. OS:centOS6.x或redhat6.x，本文采用CentOS6.5进行配置
  2. Apache-httpd-2.2.15（yum方式安装）
  3. Mysql-5.6.10（rpm方式安装）
  4. php-5.6.15（编译方式安装）

安装方式有点乱是因为先有了熟悉的httpd和mysql安装配置经验和工具，复用了。php配置比较少，只有通过网络收集资料然后实践整理。

OS基本环境要求：
openssl，openssl-devel,gcc

# 检查gcc等组件
rpm -q gcc
gcc-4.4.7-3.el6.x86_64
# 如果没有安装，一般采用yum方式安装最简单
yum search gcc|openssl|openssl-devel
yum install ...    

# apr/apr-util >= 1.2,如果apr/apr-util不存在可以再后续的安装包中先安装。
# 如果版本小于1.2，则需要先安装APR和APR-util
rpm -q apr
apr-1.2.7-11

2.安装mysql

mysql的安装可以使用yum方式安装，也可以自己下载二进制包安装，因我这里有下载好的熟悉的版本安装包和自动安装脚本，故采用后者。

linux环境安装配置工具:env-setup。下载地址在SVN下:
http://acooly.gicp.net/svn/framework/acooly-script/env-setup

直接在linux上使用svn checkout

svn checkout http://acooly.gicp.net/svn/framework/acooly-script/env-setup

你必须是我们团队成员并有密码哈。

进入env-setup/bin目录，运行：

./install.mysql.sh

请根据提示进行操作。该脚本会自动从acooly文件服务器下载安装文件，文件服务器的认证需要SVN账户认证。

该脚本功能如下：

  • 自动下载mysql的server和client安装文件，版本号：5.6.10
  • 自动卸载本机默认mysql相关的lib包，防止冲突（mysql-libs）
  • 自动安装mysql的server和client
  • 迁移主配置文件到/etc/my.cnf
  • 迁移数据目录到/data/mysql目录
  • 自动生成新密码列表（12个随机字符），修改root密码（交互式）

3.apache安装

参考：http://acooly.iteye.com/blog/1347749

采用yum方式安装。

yum intall httpd
# 安装devel是为了安装后续需要的apxs
yum install httpd-devel

安装完成后，apache的所有配置在/etc/httpd下面。主配置文件conf/httpd.conf,其他模块配置文件放到conf.d/,默认情况下只有welcome.conf

注意：

安装后，后续PHP需要的apxs文件在:/usr/sbin/apxs

mpm配置

httpd -l
Compiled in modules:
  core.c
  prefork.c
  http_core.c
  mod_so.c

输出里面的prefork.c表示采用perfork方式，那么，我需要配置conf/httpd.conf中的perfork段。

<IfModule prefork.c>
StartServers       8
MinSpareServers         5
MaxSpareServers     20
ServerLimit       1000
MaxClients       1000
MaxRequestsPerChild  4000
</IfModule>
  • StartServers: 服务器初始进程数
  • MinSpareServers: 最小空闲进程数量
  • MaxSpareServers: 最大空闲进程数量
  • ServerLimit：服务器最大进程数
  • MaxClients：最大客户请求数
  • MaxConnectionsPerChild: 每个子进程可处理的请求数。每个子进程在处理了“MaxRequestsPerChild”个请求后将自 动销毁。0意味着无限，即子进程永不销毁。虽然缺省设为0可以使每个子进程处理更多的请求，但如果设成非零值也有两点重要的好处：
    1. 可防止意外的内存泄漏。
    2. 在服务器负载下降的时侯会自动减少子进程数。

4.PHP安装

文件：

wget http://cn2.php.net/distributions/php-5.6.15.tar.gz

或：
wget –ask-password http://acooly.wicp.net/project/software/php-5.6.15.tar.gz

依赖：

yum install libxml2 -y
yum install libxml2-devel -y

安装：

./configure --prefix=/usr/local/php --with-config-file-path=/usr --with-mysql --with-apxs2=/usr/sbin/apxs --enable-sockets
  • –prefix=/usr/local/php : php的安装目录
  • –with-config-file-path=/usr： 指定默认的配置文件读取目录，如果MYSQL采用RPM方式安装，则需要设置为/usr
  • –with-mysql: 打开mysql支持
  • –with-apxs2=/usr/sbin/apxs：整合 apache，apxs功能是使用mod_so中的LoadModule指令，加载指定模块到 apache，要求 apache 要打开SO模块
  • –enable-sockets：打开 sockets 支持

配置

PHP安装完后，因为启用了apxs，所以会自动增加loadModule php…到httpd.conf文件中。原来的httpd.conf会备份为：httpd.conf.bak

建立网站文件主目录：

# mkdir /opt/www
# chown -R apache:apache /opt/www

使用vi或vim编辑httpd.con文件。进行调整配置如下：

# 配置ServerName,防止启动告警找不到domain
ServerName localhost:80

# 修改主目录：
DocumentRoot "/opt/www"

# 增加php文件的match配置：
<FilesMatch \.php$>
    SetHandler application/x-httpd-php
</FilesMatch>

<FilesMatch "\.ph(p[2-6]?|tml)$">
    SetHandler application/x-httpd-php
</FilesMatch>

<FilesMatch "\.phps$">
    SetHandler application/x-httpd-php-source
</FilesMatch>

# 修改目录默认文件:
DirectoryIndex index.html index.php

# 调整目录权限：
<Directory />
    Options FollowSymLinks
    AllowOverride None
    Order deny,allow
    Deny from all
</Directory>

<Directory "/opt/www">
    Options Indexes FollowSymLinks
    AllowOverride None
    Order allow,deny
    Allow from all
</Directory>

测试：

启动apache服务器

service httpd start

创建PHP测试页面

cd /opt/www
echo "<?php phpinfo(); ?>" > test.php
chown apache:apache test.php

访问test.php页面

curl http://localhost/test.php

安装DZ3.1

文件

  • 版本：discuz 3.1.x
  • 版本论坛：http://www.discuz.net/forum.php?mod=collection&action=view&ctid=186
  • 文件：Discuz_X3.1_SC_UTF8.zip

安装

安装前置条件为LAMP环境已搭建OK。

mkdir -p /opt/dz
mv Discuz_X3.1_SC_UTF8.zip /opt/dz
cd /opt/dz
unzip Discuz_X3.1_SC_UTF8.zip
cp -r upload/* /opt/www
chown -R apache:apache /opt/www

配置

打开浏览器，输入服务器IP（端口默认80）根据向导完成配置。一般需要配置的就是数据库相关的配置。
